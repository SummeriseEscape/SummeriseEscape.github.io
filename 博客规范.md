# Starry Breeze — GitHub Pages 文艺构成主义博客设计规范（Vibe Coding Edition）

Version: 2.0  
Framework Target: GitHub Pages  
Recommended Stack: Astro + TailwindCSS + MDX  
Design Direction: Constructivism × Summer Breeze × Starry Night × Japanese Editorial Minimalism

---

# 一、项目定义（Project Definition）

## 1.1 项目定位

这是一个：

- 文艺型个人博客
- 数字诗集
- 情绪记录空间
- 视觉化日记
- 星空与夏日主题网站

网站目标不是：

- 内容门户
- 商业资讯站
- SaaS 产品官网

而是：

“一个拥有呼吸感与情绪氛围的数字空间”。

---

# 二、Vibe Coding 核心原则（极重要）

开发过程中必须始终遵守以下原则：

---

## 2.1 网站不是“工具”

网站更像：

- 一本夏夜杂志
- 一个安静展览
- 一张深夜播放的唱片
- 一片漂浮于夜空中的页面

因此：

UI 必须服务于情绪。

---

## 2.2 页面必须有“空气”

必须：

- 大留白
- 稀疏布局
- 缓慢节奏
- 轻量信息密度

避免：

- 信息焦虑
- 功能堆叠
- 高频交互
- UI 噪音

---

## 2.3 动效必须“像风”

所有动画都必须：

- 缓慢
- 轻盈
- 有呼吸感
- 具有漂浮趋势

不要：

- 强运动
- 弹性动画
- 高频动画
- 赛博科技感

---

## 2.4 页面必须具有“静”

页面打开后：

用户首先感受到：

- 风
- 空气
- 留白
- 夏夜
- 星空

然后才阅读内容。

---

# 三、GitHub Pages 技术实现规范

# 3.1 推荐技术栈

## 推荐（最佳方案）

```bash
Astro
TailwindCSS
MDX
TypeScript
GSAP
Lenis
```

---

## 原因

Astro 非常适合：

- 内容型网站
- 静态生成
- GitHub Pages
- Markdown 驱动
- 极简性能优化

同时：

- 动画控制优秀
- 页面切换自然
- SEO 友好
- 图片优化方便

---

# 3.2 GitHub Pages 部署要求

## 必须：

- 完全静态化
- 无服务端依赖
- 所有页面 SSG
- 支持自定义域名

---

## Astro 配置要求

```js
output: "static"
```

---

## 部署目录

```bash
dist/
```

---

## GitHub Actions 自动部署

必须包含：

- 自动 build
- 自动 deploy
- branch: main

# 五、构成主义规范（Constructivism）

# 5.1 页面结构原则

页面必须：

- 几何化
- 模块化
- 留白化
- 非对称化

---

# 5.2 几何元素

必须长期存在：

- 圆形
- 细线
- 点阵
- 网格
- 半透明矩形
- 几何切割

---

# 5.3 几何风格

所有几何：

- 不可过重
- 不可高对比
- 不可赛博朋克

而应：

- 柔和
- 空气化
- 半透明
- 具有漂浮感

---

# 5.4 视觉流向

整个页面的视觉运动方向：

```text
左上 → 右下
```

营造：

- 风吹感
- 漂浮感
- 时间流动感
